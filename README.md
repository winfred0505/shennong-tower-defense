# 毒劇中藥安全性網頁塔防遊戲 成果報告 (Walkthrough)

本專案已順利完成實作，並成功將簡報「2025毒劇中藥材安全性」中的核心概念，融入為一款寓教於樂的橫向捲軸塔防網頁遊戲原型，命名為**「神農禦邪傳：毒劇中藥安全性守護戰」**。

---

## 實作內容與亮點

本遊戲完全基於 **HTML5, CSS3, Vanilla JavaScript (Canvas 2D)** 實作，整合於單一檔案中，無須任何外部依賴或圖片載入，具有極高的效能與跨平台兼容性。

### 1. 遊戲核心機制
*   **答題經濟系統**：
    *   包含 **30 道精選中藥安全性選擇題**，涵蓋基原、十八反、十九畏、巴豆/半夏/馬錢子減毒解毒、藥事法規、臨床給藥查核等，大幅增加多樣性。
    *   答對立即獎勵 **+100金幣**；答錯鎖定 3 秒冷卻並顯示**中藥安全性分析解析**，且將**正確的選項以黃色高亮顯示**。
    *   被動金幣加成極低（每秒 +1金幣），促使玩家專注答題。
*   **召喚與戰鬥 AI**：
    *   4 種正義中藥衛士：
        1.  🧑‍🌾 **赭鞭藥童**：近戰檢驗與快速打擊。
        2.  🧙‍♂️ **炮製藥師**：投擲滾燙藥罐造成範圍傷害，具備「高溫水解生物鹼」減速 30% 特效。
        3.  🛡️ **解毒綠豆兵**：綠豆甘草化身，超高 HP 抵擋攻擊，被動減少同伴負面狀態。
        4.  👑 **神農化身**：對抗 BOSS 的核心，具備神農光環持續治療周邊隊友。
    *   4 種病邪敵人：
        1.  👹 **生川烏兵**：雙酯生物鹼劇毒，近戰高輸出。
        2.  🐗 **生巴豆獸**：巴豆油致瀉毒素，造成持續扣血（中毒狀態）。
        3.  👾 **生馬錢子怪**：番木鱉鹼神經毒，射出子彈使友軍短暫麻痺（眩暈狀態）。
        4.  🗿 **硃砂禁忌魔 (BOSS)**：重金屬汞害禁用硃砂，高 HP 與超高破壞力。
*   **精美與極簡佈局優化 (新要求)**：
    *   **雙欄並排版面**：將原先垂直堆疊的「答題區」與「控制區」改為左右並排（左側 60% 答題，右側 40% 召喚），大幅縮短垂直高度。
    *   **頂部 HUD 整合**：將防守方與敵方城堡的血量條移至頂部 HUD 資訊列，使主要戰鬥狀態一目了然。
    *   **2x2 召喚卡片設計**：將原本佔用整排的兵種卡片改為 2x2 緊湊網格，並採橫向精緻排版（左圖右文），節省空間且極具質感。
    *   **互動說明彈窗**：移除底部冗長的兵種靜態介紹，改為頂部 `❓ 說明` 按鈕，點擊彈出遮罩視窗，達到單一視窗（無滾動條）的完整遊戲體驗。
*   **開始與歡迎畫面 (新要求)**：
    *   新增「開始遊戲歡迎畫面」遮罩，載入時不自動運行遊戲，讓玩家閱讀完玩法規則、獲取金幣機制及波次說明後，點擊「開始遊戲」按鈕才啟動遊戲引擎與出題。

## 驗證與測試結果

為確保遊戲品質，我們在本地環境進行了多項自動與手動測試，並在測試中修正了關鍵運行錯誤：

1.  **HTML 語意與結構驗證**：
    *   使用 Python 的 HTML 剖析模組進行靜態結構檢查，確認標籤完整閉合，且 DOM 物件命名具備唯一性，測試**通過**。
2.  **JavaScript 語法與邏輯編譯驗證**：
    *   使用 Node.js `vm` 模組在本地執行 JavaScript 靜態語法檢查，無任何語法錯誤，測試**通過**。
3.  **運行期 DOM 環境模擬與 Bug 修正 (重要)**：
    *   **發現問題**：在首次啟動時，瀏覽器發生 `ReferenceError: Cannot access 'playerCastle' before initialization` 錯誤，導致腳本中斷、畫面呈現全黑且卡在「載入題目中...」。此因 `resizeCanvas()` 被立即呼叫，而其內參照的變數 `playerCastle` 與 `enemyCastle` 宣告在後，觸發了 ES6 暫時性死區 (TDZ)。
    *   **修復方案**：將城堡及實體變數的 `let` 宣告移至腳本最頂部（`resizeCanvas()` 定義之前）。經 Node.js 模擬 DOM 執行測試，確認已**完全修復**，成功進入遊戲循環。
4.  **解析度與佈局響應測試**：
    *   調整視窗大小時，Canvas 的寬高自動縮放，且 castles 和 units 位置隨 resize 事件重新估算，避免了漂浮、出界或重疊，測試**通過**。
5.  **答題冷卻機制測試**：
    *   答錯時，冷卻計時器與動畫條正確倒數 3 秒，期間點擊選項無效，3秒後載入新題目；答對時，金幣即時增加 100 並播放飄字動畫，測試**通過**。
